# <a name="microsoft-project-service-description"></a>Microsoft Project – Dienstbeschreibung

## <a name="overview"></a>Übersicht

Microsoft Project bietet die folgenden Anwendungen, um die Anforderungen Ihres Unternehmens an das Projekt- und Arbeitsmanagement zu erfüllen:

- Project für das Web
- Project Online 
- Project Online Desktop-Client

Dieser Artikel hilft Ihnen zu verstehen, welche Anwendungen in jedem abonnement Project bereitgestellt werden und welche Funktionen jede Anwendung bietet.

### <a name="project-for-the-web"></a>Project für das Web

Project für das Web ist Microsofts neuestes Angebot für Cloud-basiertes Arbeiten und Projektmanagement. Project für das Web bietet einfache, leistungsstarke Arbeitsverwaltungsfunktionen, um die meisten Anforderungen und Rollen zu erfüllen. Project Manager und Teammitglieder können Project für das Web verwenden, um Arbeiten jeder Größe zu planen und zu verwalten.

Project für das Web basiert auf der Microsoft Power Platform. Die Power Platform besteht aus PowerApps, Power Automate, Power BI und Microsoft Dataverse. Project für die Webdaten werden im Microsoft Dataverse gespeichert. 

Project für das Web ist über drei Abonnements verfügbar: Project Plan 1, Project Plan 3 und Project Plan 5. Features, die in [Project für die Webfeatures](#project-for-the-web-features) aufgeführt sind, interagieren nur mit Daten im Project für den Webdatenspeicher.

### <a name="project-online"></a>Project Online
 
Project Online ist eine flexible Online-Lösung für Project PortfolioManagement (PPM) und den Arbeitsalltag. Project Online bietet leistungsstarke Projektmanagement-Funktionen für die Planung, Priorisierung und Verwaltung von Projekten und Projektportfolioinvestitionen &mdash; von fast überall auf fast jedem Gerät. Project Online kann von Administratoren, Portfoliomanagern, Portfoliobetrachtern, Projektmanagern, Ressourcenmanagern, Teamleitern und Teammitgliedern verwendet werden.

Project Online basiert auf der SharePoint Plattform und speichert Daten im SharePoint Datenspeicher.

Project Online ist über zwei Abonnements erhältlich: Project Plan 3 und Project Plan 5. Features, die in [Project Online Features](#project-online-features) aufgeführt sind, interagieren nur mit Daten im Project Online SharePoint Datenspeicher.

Die Teammitgliederfunktionalität für Project Online ist über ein Project Online Essentials- oder Project Plan 1-Abonnement verfügbar. Project Online Essentials ist ein *Teammitglieder-Add-On-Abonnement* für Kunden, die über Project Plan 3 oder Project Plan 5 Abonnements verfügen. Project Online Essentials und Project Plan 1 Abonnements bietet Teammitgliedern eine Weboberfläche zum Ausführen von Aufgaben- und Zeitaktualisierungen für Projekte, die in Project Online verwaltet werden.

Teammitglieder mit Project Online Essentials oder Project Plan 1 Abonnements können die folgenden Project Online Funktionen ausführen:

- Verwenden einer webbasierten Schnittstelle
- Aktualisieren von Aufgaben, Problemen und Risiken
- Übermitteln von Arbeitszeittabellen
- Freigeben von Dokumenten und Zusammenarbeit mit Microsoft Teams oder Skype for Business Benutzern

Project Online Essentials enthält keine Project für das Web. Project Plan 1 umfasst sowohl Project für das Web als auch für Project Online Essentials.

### <a name="project-online-desktop-client"></a>Project Online Desktop-Client

Microsoft Project Online Desktop-Client ist ein Projektmanagementprogramm, das automatisierte Planung, Projektressourcenmanagement und integrierte Sanfangerstellung bietet. Project Online Desktopclient kann als eigenständige Anwendung oder als Project Online verwendet werden. Project Online Desktopclient ist die Abonnementversion des Project Professional Desktopclients und in Project Plan 3 und Project Plan 5 Abonnements verfügbar.

### <a name="using-project-for-the-web-and-project-online-together"></a>Project für das Web und Project Online zusammen verwenden

Obwohl Project für das Web und Project Online separate Anwendungen sind, können Sie sie nebeneinander verwenden. Sie können alle Ihre Projekte aus beiden Anwendungen in Project Home sehen. Sie können die Project Roadmap-Funktion verwenden, um Roadmaps zu erstellen, die Projekte aus beiden Anwendungen enthalten. Und Sie können Power BI und die Project für das Web-Power BI-Inhaltspaket verwenden, um Einblicke in Projekte und Ressourcen in den beiden Anwendungen zu erhalten. <sup>3</sup>

Dieses Diagramm zeigt, wie die Anwendungen nebeneinander zusammenarbeiten können.

![Diagramm mit Project im Web und Project Online zusammen](../media/project-online-applications.png)

In Zukunft wird sich Microsoft auf Project für die Web-Innovationen konzentrieren. Sie können Project Online jedoch weiterhin vertraulich verwenden, da sie weiterhin wichtige Leistungs- und Sicherheitsverbesserungen erhalten.

## <a name="service-considerations"></a>Service-Überlegungen

> [!NOTE]
> Project für das Web ist derzeit für GCC, GCC High und DoD nicht verfügbar. Wir arbeiten daran, Ihnen Project für das Web bereitzustellen, sind aber derzeit nicht in der Lage, einen festen Zeitrahmen zu teilen.

### <a name="licensing-considerations"></a>Hinweise zur Lizenzierung

- Jede Interaktion auf einer Project Online Website erfordert mindestens eine Project Plan 3 oder Project Plan 5 Abonnement innerhalb des Mandanten.
- Wenn Ihr letztes Project Plan 1, Project Plan 3 oder Project Plan 5 Abonnement abläuft, wird Ihr Project für die Webinstanzen erst automatisch gelöscht, wenn Sie keine aktiven Abonnements haben, die von Microsoft Dataverse abhängen.
- Wenn Ihr letztes Project Plan 3 oder Project Plan 5 Abonnement abläuft, werden Ihre Project Online-Instanzen nach 120 Tagen gelöscht.
- Bei Project Online Testabonnements werden Ihre Testinstanzen 30 Tage nach Ablauf des Testzeitraums gelöscht.
- Bei Project für die Webtestabonnements werden Ihre Testinstanzen erst gelöscht, wenn Sie keine aktiven Abonnements haben, die von Microsoft Dataverse abhängen.

### <a name="use-of-sharepoint-online"></a>Verwenden von SharePoint Online

Project Online erfordert die Nutzung von SharePoint Online, das als Teil von Project Online vorgesehen ist. Die Rechte an der SharePoint Online-Funktionalität, die mit Project Plan 3 oder Project Plan 5 Abonnements bereitgestellt wird, beschränken sich auf das Speichern und Zugreifen auf Daten, um Project Online zu unterstützen.

### <a name="project-roadmap-and-power-automate"></a>Project Roadmap und Power Automate

Project Die Roadmap erfordert die Verwendung von Power Automate, das als Teil Ihres Project-Abonnements bereitgestellt wird. Die Rechte an Power Automate Funktionalität sind auf die Power Automate Funktionen beschränkt, die für Project Roadmap erforderlich sind. Power Automate für Project Roadmap erforderliche Funktionalität wird im Abschnitt **Apps** des Microsoft 365-Admincenters als **Datenintegration für Project mit Flow** Dienstplan angezeigt.

### <a name="project-for-the-web-and-microsoft-dataverse"></a>Project für das Web und Microsoft Dataverse

Project für das Web erfordert die Verwendung des Microsoft Dataverse zum Speichern seiner Daten. Eine Microsoft Dataverse-Datenbank wird als Teil Ihres Project-Abonnements bereitgestellt. Die Rechte an Microsoft Dataverse-Funktionen sind auf das Speichern und Zugreifen auf Daten beschränkt, um Project für das Web zu unterstützen. Die von Project erforderliche Microsoft Dataverse-Funktionalität wird im Abschnitt **Apps** des Microsoft 365 Admin Centers als **Common Data Service for Project** Dienstplan angezeigt.

Für Project Kunden mit fünf (5) oder mehr Project für die Weblizenzen können Project für Webkunden in Power Platform Production- und Sandbox-Umgebungen bereitgestellt werden.
